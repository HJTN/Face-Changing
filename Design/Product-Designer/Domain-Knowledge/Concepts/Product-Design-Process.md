# Product Design Process

## Overview
프로덕트 디자이너는 아이디어에서 출시까지 제품의 전체 설계 과정을 이끈다. 단순 UI 작업이 아닌, 비즈니스 목표·사용자 니즈·기술 제약 사이의 균형을 찾는 과정이다.

## 더블 다이아몬드 모델 (Double Diamond)

`
발산                수렴             발산               수렴
[Discover] ──────→ [Define] ──────→ [Develop] ────→ [Deliver]
 문제 공간 탐색      진짜 문제 정의    해결책 아이디에이션   최적 솔루션 구현
`

## 프로세스 단계

### 1. Discovery
- 사용자 인터뷰, 시장 조사
- 경쟁사 분석, Analytics 검토
- 산출물: Research Report, Persona

### 2. Definition
- HMW(How Might We) 질문 도출
- 문제 우선순위 결정
- 산출물: Problem Statement, Jobs-to-be-Done

### 3. Design
- 정보 아키텍처, User Flow
- Wireframe → Mockup → Prototype
- 산출물: Figma 파일, 프로토타입

### 4. Testing
- 사용성 테스트, A/B 테스트
- 반복 수정
- 산출물: 테스트 결과 보고서

### 5. Handoff & Launch
- 개발자와 스펙 공유 (Zeplin, Figma Dev Mode)
- QA 지원
- 산출물: 최종 디자인 명세, 론치 노트
