# Design Thinking

## Overview
디자인 씽킹(Design Thinking)은 복잡한 문제를 창의적·실험적으로 해결하는 사고방식이자 프로세스다. 기술적 실현 가능성(Feasibility), 비즈니스 생존 가능성(Viability), 인간적 바람직함(Desirability)의 교차점에서 솔루션을 찾는다.

## 5단계 프로세스

| 단계 | 핵심 질문 | 산출물 |
|---|---|---|
| **Empathize** | 사용자는 무엇을 경험하는가? | 인터뷰 노트, 관찰 기록 |
| **Define** | 진짜 문제는 무엇인가? | HMW(How Might We) 질문, POV 문장 |
| **Ideate** | 어떤 해결책이 가능한가? | 아이디어 목록, 마인드맵 |
| **Prototype** | 아이디어를 빠르게 구현할 수 있는가? | 종이 프로토타입, 와이어프레임 |
| **Test** | 솔루션이 문제를 해결하는가? | 사용성 테스트 결과, 피드백 |

## 핵심 마인드셋
- **인간 중심**: 기술이 아닌 사람에서 출발
- **실험 지향**: 빠른 실패가 늦은 성공보다 낫다
- **협업**: 다양한 배경의 팀원이 함께 발산·수렴
- **낙관주의**: 모든 문제에는 해결책이 있다는 믿음
