# Human-Centered Design

## Overview
인간 중심 설계(HCD)는 사용자의 니즈·능력·행동을 설계 과정의 중심에 놓는 철학이자 방법론이다. IDEO와 스탠퍼드 d.school이 보급한 접근법으로 ISO 9241-210 국제 표준으로도 정의되어 있다.

## 4가지 핵심 원칙 (Don Norman)

1. **사람 중심** — 기술이나 비즈니스보다 사람의 니즈를 먼저 이해한다
2. **반복적 개선** — 한 번에 완벽한 답을 찾으려 하지 않고 프로토타입→테스트→개선을 반복한다
3. **협력** — 다학제 팀(디자이너·개발자·연구자·사용자)이 함께 문제를 해결한다
4. **전체론적 관점** — 개별 기능이 아닌 전체 경험(End-to-End)을 바라본다

## HCD 프로세스
`
공감 (Empathize) → 정의 (Define) → 아이디어 (Ideate) → 프로토타입 (Prototype) → 테스트 (Test)
`

## Why It Matters
- HCD를 적용한 제품은 사용자 만족도·전환율·재방문율이 높다
- 개발 후반 수정 비용을 줄인다 (문제를 초기에 발견)
- 접근성(Accessibility)과 포용적 디자인의 기반이 된다
