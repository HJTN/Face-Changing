# Product Discovery

## Overview
프로덕트 디스커버리(Product Discovery)는 만들어야 할 것이 무엇인지를 결정하는 과정이다. 올바른 것을 만들기 위한 검증 활동으로, 빌드 전에 리스크(가치·사용성·실현 가능성·비즈니스 생존 가능성)를 줄인다.

## 4가지 리스크 (Marty Cagan — INSPIRED)

| 리스크 | 질문 | 검증 방법 |
|---|---|---|
| **Value Risk** | 고객이 원하는가? | 인터뷰, 설문, 데이터 분석 |
| **Usability Risk** | 사용자가 사용할 수 있는가? | 사용성 테스트 |
| **Feasibility Risk** | 기술적으로 만들 수 있는가? | 기술 스파이크, PoC |
| **Business Viability Risk** | 비즈니스가 성립하는가? | 수익 모델, 법적·마케팅 검토 |

## 디스커버리 프로세스

### 더블 다이아몬드 응용

`
Discover(발견) → Define(정의) → Ideate(아이디에이션) → Prototype(프로토타입) → Test(검증)
`

### OKR → 기회 → 솔루션

`
OKR: "신규 사용자 전환율 20% 향상"
    ↓
기회 트리: "온보딩 첫날 이탈 50% 발생"
    ↓
솔루션: A. 온보딩 단계 3개로 줄이기
        B. 인앱 가이드 투어 추가
        C. 이메일 드립 캠페인
`

## 가설 검증 기법

### 사용자 인터뷰 원칙
- 솔루션 소개 X → 문제 탐색에 집중
- 과거 행동 질문: "마지막으로 [문제 상황]이 있었을 때 어떻게 하셨나요?"
- "만약 이 기능이 있다면 쓰시겠어요?" → 가장 쓸모없는 질문

### Fake Door Test
- 아직 만들지 않은 기능 버튼을 UI에 추가
- 클릭률로 수요 측정
