# Risk Management in Technical Projects

## Overview
리스크 관리는 프로젝트 목표 달성을 방해할 수 있는 불확실성을 사전에 식별·분석·대응하는 체계적 프로세스다. 기술 프로젝트에서 리스크는 기술·일정·자원·비용·요구사항 등 다양한 측면에서 발생한다.

## 리스크 관리 프로세스

`
1. 리스크 식별  → 리스크 레지스터 작성
2. 정성적 분석  → 발생 가능성 × 영향도 = 리스크 점수
3. 정량적 분석  → 금전적 영향 계산 (필요시)
4. 대응 계획    → 전략 선택
5. 모니터링·통제 → 리스크 상태 지속 추적
`

## 리스크 평가 매트릭스

`
         낮음   중간   높음
높음  │  중    높    매우높음
영향  │
중간  │  낮    중    높
      │
낮음  │  매우낮 낮   중
      └───────────────────
         낮음   중간  높음
              발생 가능성
`

## 리스크 대응 전략

| 전략 | 설명 | 예시 |
|---|---|---|
| **회피 (Avoid)** | 리스크 원인 제거 | 위험한 기술 사용 안 함 |
| **전가 (Transfer)** | 제3자에게 이전 | 보험, 계약 |
| **완화 (Mitigate)** | 발생 가능성·영향 감소 | 프로토타입, PoC |
| **수용 (Accept)** | 발생 시 처리 | 비상 예산 확보 |

## 기술 프로젝트 공통 리스크

| 리스크 | 발생 원인 | 대응 |
|---|---|---|
| 요구사항 변경 | 이해관계자 합의 미흡 | 변경 관리 프로세스, 우선순위 동결 |
| 기술 부채 | 일정 압박으로 인한 단기 해결 | Sprint 내 기술 부채 할당 |
| 핵심 인력 이탈 | 지식 편중 | 문서화, 지식 공유 |
| 범위 크리프 | 요구사항 증가 | 변경 관리 거버넌스 |
| 통합 오류 | 외부 시스템 연동 | API 계약 테스트, Mock |
