# Project Management Fundamentals

## Overview
기술 프로젝트 관리는 소프트웨어·시스템 개발 프로젝트를 계획·실행·통제·종료하는 체계적 방법론이다. 기술적 이해와 관리 능력을 동시에 요구하는 역할이다.

## 프로젝트 제약 삼각형 (Triple Constraint)

`
         범위 (Scope)
            /\
           /  \
          /    \
         /  품질  \
        /  (Quality) \
       ────────────────
    시간 (Time)    비용 (Cost)
`

세 꼭지점을 동시에 최적화할 수 없다. 하나를 변경하면 다른 하나 이상에 영향.

## 프로젝트 관리 방법론

### Waterfall (폭포수)
- 요구사항 → 설계 → 개발 → 테스트 → 배포 → 유지보수
- 각 단계가 완전히 완료된 후 다음 단계 진행
- 적합: 요구사항이 명확하고 변경이 없는 프로젝트

### Agile
- 반복적·증분적 개발, 변화에 유연하게 대응
- 스크럼, 칸반, SAFe 등 다양한 구현 방식

### 하이브리드
- Waterfall의 계획성 + Agile의 유연성 결합
- 대규모 엔터프라이즈 프로젝트에서 많이 사용
