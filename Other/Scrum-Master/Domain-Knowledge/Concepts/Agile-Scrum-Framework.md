# Agile Scrum Framework

## Overview
스크럼(Scrum)은 복잡한 제품 개발을 위한 경량 애자일 프레임워크다. 2020년 스크럼 가이드는 스크럼을 "팀, 이벤트, 아티팩트, 그 사이에 놓인 규칙으로 구성된 프레임워크"로 정의한다.

## 스크럼 3가지 기둥 (Empiricism)

`
투명성 (Transparency) + 검사 (Inspection) + 적응 (Adaptation)
= 경험주의적 접근 (예측이 아닌 경험으로 제어)
`

## 스크럼 3가지 역할

| 역할 | 책임 |
|---|---|
| **Product Owner** | 제품 백로그 관리, ROI 극대화, 이해관계자 관리 |
| **Scrum Master** | 스크럼 코치, 장애물 제거, 프로세스 개선 |
| **Developers** | 스프린트 목표 달성, Done 정의 구현 |

## 스크럼 5가지 이벤트

| 이벤트 | 목적 | 시간 박스 |
|---|---|---|
| **Sprint** | 1-4주 개발 주기 | 1-4주 |
| **Sprint Planning** | 스프린트 목표·작업 계획 | Sprint 주수 × 2시간 |
| **Daily Scrum** | 일일 계획 동기화 (진행/장애/계획) | 15분 |
| **Sprint Review** | 스프린트 결과물 검사·적응 | Sprint 주수 × 1시간 |
| **Sprint Retrospective** | 팀 프로세스 개선 계획 | Sprint 주수 × 45분 |

## 스크럼 3가지 아티팩트

| 아티팩트 | 내용 | 책임자 |
|---|---|---|
| **Product Backlog** | 제품에 필요한 모든 작업 목록 | Product Owner |
| **Sprint Backlog** | 스프린트에 선택된 작업 + 계획 | Developers |
| **Increment** | 완료된 작업의 합 (Done 기준 충족) | Developers |

## Definition of Done (완료 기준)
- 팀이 합의한 완료 기준
- 예: 코드 리뷰 완료 + 단위 테스트 통과 + QA 검증 + 문서 업데이트
