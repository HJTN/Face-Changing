# Scrum Values and Pillars

## Overview
스크럼의 5가지 가치와 3개의 기둥은 단순한 도구 사용 지침이 아닌, 팀이 일하는 방식의 철학적 기반이다. 프로세스 준수 이전에 이 가치와 기둥을 내면화해야 스크럼이 제대로 작동한다.

## 스크럼 5가지 가치 (5 Values)

### 1. 용기 (Courage)
- 어려운 문제를 직면할 용기
- "이 스프린트에 완료할 수 없습니다"라고 솔직하게 말하는 용기
- 레거시 시스템의 문제를 공개적으로 제기하는 용기

### 2. 집중 (Focus)
- 스프린트 목표에 집중, 외부 방해 차단
- WIP(Work in Progress) 제한으로 여러 일을 동시에 하지 않음

### 3. 헌신 (Commitment)
- 팀 전체가 스프린트 목표에 헌신
- 개인의 완료가 아닌 팀의 완료

### 4. 존중 (Respect)
- 팀원의 의견·역량·경험 존중
- 실패로부터 비난하지 않음

### 5. 개방성 (Openness)
- 작업·진행 상황·어려움을 투명하게 공유
- 이해관계자에게도 솔직하게 현황 보고

## 경험주의 3가지 기둥

### 투명성 (Transparency)
- 모든 이해관계자가 프로세스와 작업 현황을 볼 수 있음
- 스프린트 백로그, 번다운 차트, DoD 공개

### 검사 (Inspection)
- 스크럼 이벤트에서 아티팩트와 진행 상황을 자주 검사
- 데이터 기반 현실 인식 (희망이 아닌 사실)

### 적응 (Adaptation)
- 검사 결과 문제가 발견되면 즉시 조정
- 스프린트 레트로스펙티브가 가장 중요한 적응 기회

## 스크럼 마스터의 역할
- 가치와 기둥을 팀에 가르치고 실천하게 도움
- 스크럼이 올바르게 이해되고 실행되는지 보장
- 장애물(Impediment) 제거로 팀의 잠재력 최대화
