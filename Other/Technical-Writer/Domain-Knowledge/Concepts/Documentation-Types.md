# Documentation Types

## Overview
기술 문서는 독자와 목적에 따라 여러 유형으로 나뉜다. Diátaxis 프레임워크는 모든 기술 문서를 4가지 유형으로 분류해 각각의 역할과 작성 방법을 명확히 한다.

## Diátaxis 프레임워크 (Daniele Procida)

`
         학습 지향                  작업 지향
┌─────────────────────────────────────────────┐
│  TUTORIAL          │  HOW-TO GUIDE           │
│  (배우기)           │  (해결하기)              │
│  학습 여정을 안내함  │  특정 문제 해결 방법     │
├─────────────────────┼─────────────────────────┤
│  EXPLANATION       │  REFERENCE               │
│  (이해하기)         │  (정보 찾기)             │
│  개념·배경 설명     │  정확한 기술 명세        │
└─────────────────────┴─────────────────────────┘
   이해 지향                   정보 지향
`

## 4가지 문서 유형

### Tutorial (튜토리얼)
- 목적: 처음 배우는 사람이 뭔가를 성공적으로 완료하게 함
- 형식: 단계별 실습 ("이것을 하세요")
- 성공 기준: 독자가 작동하는 결과물을 만들어냄
- 예: "첫 번째 API를 만들어보세요"

### How-To Guide
- 목적: 특정 목표 달성 방법 안내
- 형식: 특정 문제 해결 단계
- 가정: 독자가 기본 지식 보유
- 예: "JWT 인증 추가하는 방법"

### Reference
- 목적: 정확한 기술 정보 제공
- 형식: 완전하고 정확한 명세 (API 레퍼런스, 설정 옵션)
- 코드처럼 정확해야 함
- 예: "API 엔드포인트 목록", "설정 파라미터"

### Explanation
- 목적: 개념·배경·설계 이유 이해
- 형식: 산문 형식의 설명
- "왜"에 답함
- 예: "우리가 마이크로서비스를 선택한 이유"
